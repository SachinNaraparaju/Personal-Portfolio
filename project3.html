    Project Purpose and Goal

    The purpose of the "Personal Budget" app is to develop a user-friendly and professional-looking budgeting tool that fulfills the creator's specific requirements for effectively tracking expenses. By incorporating features for managing multiple budgets and categorizing expenses accordingly, the app aims to provide a seamless experience for users to monitor their financial activities. The primary goal is to offer a simple and efficient user experience, ensuring smooth data management and allowing users to easily view their remaining budget and make adjustments by adding or removing expenses as needed.

    Web Stack Explanation

    The "Personal Budget" app is built using a simple web stack that emphasizes a seamless client-side experience. The front-end leverages the power of React, with a primary focus on React routing features to ensure smooth navigation and an intuitive user interface. By utilizing local storage to store data, the app prioritizes privacy and speed, offering users a secure and efficient platform to manage their budgets. In the future I plan to store the data using a database.

    The back-end of the app is powered by NodeJS, allowing for efficient handling of server-side operations. All the logic of the application is written in JavaScript, enabling a cohesive and consistent codebase throughout the project.

    Problems and Thought Process

    During the development of the "Personal Budget" app, my primary focus was on implementing the specified features, creating a seamless user experience with React routing, and ensuring the necessary logic for budget and expense management. Initially, I encountered a challenge in determining the best data storage approach. After careful consideration, I opted for local storage as it provided a suitable solution for my needs, avoiding the complexity of managing a database and safeguarding users' transaction privacy.

    One significant hurdle I faced was with website routing. I wanted to maintain consistent data on every page while also having separate pages for each new budget and its associated expenses. To address this, I devised a system involving unique IDs appended to the URL for each budget page, allowing me to achieve the desired navigation and data organization.

    Throughout the project, I honed my designing skills, improved my ability to plan and structure websites effectively, and gained proficiency in manipulating data. The process of creating the "Personal Budget" app provided invaluable experience in web development, equipping me with practical skills that will benefit future projects.
